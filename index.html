2) Fundamentals: Observability, Controllability, Testability

Controllability

Ability to force internal nodes to desired logic states through primary inputs or scan controls.

Observability

Ability to propagate an internal fault effect to a measurable point: scan output, primary output, or MISR signature.

Fault Coverage vs Defect Coverage

Fault coverage is model-based; defect coverage reflects physical defects. Both are needed for outgoing quality.

Alias Risk

In compaction, different failing responses may map to same compressed signature; architecture must keep aliasing acceptable.

4) Scan Architecture and Compression

Full Scan vs Partial Scan

Full scan improves ATPG convergence and debug; partial scan may save area/timing but reduces controllability.

Chain Balancing

Balanced chain lengths minimize shift time and power hotspots on tester.

EDT/Compression

On-chip decompressor and compactor reduce tester channels and pattern volume while preserving coverage.

X-Bounding

Control unknown sources and masking to avoid false failures and compactor corruption.

5) ATPG, Pattern Quality, and Diagnosis

Constrained ATPG

Respect clock/reset/test-mode constraints to avoid invalid states while preserving detectability.

Pattern Compaction

Static and dynamic compaction reduce vector count without degrading target fault detection.

N-Detect Strategy

Multiple detections per fault improve robustness to unmodeled defects and timing noise.

Diagnosis Loop

Fail logs -> suspect ranking -> layout correlation -> systematic root cause -> corrective action.

ATPG Quality Knob Impact Tradeoff
Higher N-detect Better defect sensitivity More patterns/test time
Tighter timing for transition tests Catches marginal delays Possible overkill if setup not correlated
Cell-aware patterning Better transistor-level defect capture Higher ATPG complexity
X-handling aggressiveness Cleaner signatures Potential masking loss if overused

6) Memory BIST, Logic BIST, and In-System Test

MBIST Essentials

March algorithms detect stuck-at, coupling, transition, and neighborhood pattern sensitive faults in SRAMs/ROMs.

Repair (BIRA/BISR)

Analyze fail maps, allocate spares, and program redundancy to recover yield.

LBIST Overview

PRPG/MISR-based pseudo-random patterning for logic self-test, often with deterministic top-up patterns.

Mission-mode Test

Periodic self-test for safety domains with bounded runtime and controlled performance impact.

8) Low-Power, Multi-Clock, and OCC-Centric DFT

Power-aware ATPG

Constrain peak and average activity in shift and capture to prevent IR-drop false failures.

Clock Domain Crossings

Handle asynchronous boundaries with proper test clocks, lockup latches, and launch/capture intent.

On-Chip Clock Control (OCC)

Generates deterministic at-speed pulses for transition/path-delay tests under tester timing limits.

Power Gating Awareness

Isolated/off domains require explicit test modes, retention handling, and safe observation points.

11) Test Economics and Program Optimization

Core Cost Equation

Cost of Test = Tester Time + Pattern Memory + Multisite Limits + Yield Loss due to escapes/overkill.

Levers

Compression ratio, pattern count, parallelism, adaptive testing, binning policy, and diagnosis loop speed.

KPIs

Coverage, DPPM, overkill rate, TTR (test time reduction), yield, and diagnosis turnaround time.

Exit Criteria

Coverage and quality targets met with controlled risk, bounded cost, and reproducible silicon correlation.

12) Practical Scenarios: What To Do and Why

Scenario A: Scan Chain Fails During Shift

Symptoms: Mismatched unload on one chain segment.

Approach: Validate mode controls, isolate chain segment, check lockup latch polarity and domain crossing.

Scenario B: Low Transition Coverage

Symptoms: Good stuck-at but weak transition/path delay detectability.

Approach: Revisit OCC pulses, launch/capture constraints, and false-path/clock-gating declarations.

Scenario C: High Unknown (X) Pollution

Symptoms: Excessive masking and unstable signatures.

Approach: Improve X-source control, add X-bounding, model RAM/analog interfaces correctly.

Scenario D: Memory Yield Drop

Symptoms: Fail clusters around certain arrays at wafer sort.

Approach: Correlate March fail maps, evaluate spare strategy, and tune repair allocation policy.

Scenario E: Board Bring-up JTAG Instability

Symptoms: Inconsistent IDCODE or boundary responses.

Approach: Check TAP timing, pull resistors, chain order, and instruction path correctness.

Scenario F: Tester Overkill Under Peak Activity

Symptoms: Random fails with poor repeatability.

Approach: Tighten power-aware fill, stagger chain activity, and correlate with power grid hotspots.
